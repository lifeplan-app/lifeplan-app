# 監査レポート：ライフイベント費用（ⓘ）

- **対象領域**: ⓘ 年次のライフイベント費用（教育費・育休収入減・住宅（ローン／賃貸）・介護費・奨学金返済）を算出する `calcLECostByYear`
- **監査日**: 2026-04-24
- **信頼度判定**: ❌ 要対応

## 1. 関数の目的と入出力

### `calcLECostByYear(year, opts)`

- **目的**: 指定暦年 `year` について、その年に実発生するライフイベント由来のキャッシュアウト（万円/年）を 5 カテゴリ別に算出し、`{ childcare, education, mortgage, care, scholarship }` オブジェクトで返す。
- **入力**:
  - `year: number`（暦年）
  - `opts: { noChild?, noLoan?, noCare?, noScholarship? }` — カテゴリ別スキップフラグ。既定はすべて `false`（全カテゴリ算入）。
  - `state.lifeEvents.{children, housingType, mortgage, rent, care, scholarships}` と `state.finance.{income, bonus}` を参照。
- **戻り値**: `{ childcare: number, education: number, mortgage: number, care: number, scholarship: number }`（各値は万円/年、整数とは限らない。`mortgage` のみ `Math.round` で整数化される箇所あり）。

## 2. 使用している計算式

### 2.1 教育費（`index.html:13063-13067`）

```js
Object.entries(PHASE_AGES).forEach(([phase, ages]) => {
  if (ages.includes(age)) {
    costs.education += EDU_COST[phase][c.phases?.[phase] || 'public'] || 0;
  }
});
```

- 数式: 子どもの年齢 `age = year - c.birthYear` がフェーズのいずれかのレンジに含まれるとき、そのフェーズのテーブル値（`EDU_COST[phase][selection]`）を当年に加算。
- **テーブル値は固定**。インフレ適用なし、学年進行の物価補正なし。
- 同一 `age` は 1 フェーズのみ該当（重なりがないよう `PHASE_AGES` が設計されている）。
- `age < 0`（未出生）は `return` でスキップ（`index.html:13019`）。
- `age >= 22` はどの `PHASE_AGES` にも含まれないため自動的に 0。

### 2.2 教育費テーブル `EDU_COST`（`index.html:12097-12104`）

```js
const EDU_COST = {
  nursery:      { public: 20,  private: 45 },   // 0-2歳
  kindergarten: { public: 6,   private: 17 },   // 3-5歳（無償化後）
  elementary:   { public: 35,  private: 167 },  // 6-11歳
  juniorhigh:   { public: 53,  private: 143 },  // 12-14歳
  highschool:   { public: 51,  private: 105 },  // 15-17歳
  university:   { national: 82, public: 93, private_lib: 115, private_sci: 154, none: 0 },
};
```

- コメント「文部科学省子供の学習費調査 令和3年度ベース」。ただし文科省調査は**幼稚園〜高校のみ**を対象としており、`nursery`（保育園）と `university` は別出典のはずだが**出典明記なし**。

### 2.3 育休収入減（`index.html:13022-13060`）

```js
const calcLeaveReduction = (person, offsetMonths) => {
  const leaveMonths = person.leaveMonths || 0;
  if (!leaveMonths) return 0;
  const startMonthFromBirth = offsetMonths;
  const endMonthFromBirth   = startMonthFromBirth + leaveMonths;
  const yearStart = age * 12;
  const yearEnd   = yearStart + 12;
  const overlapStart = Math.max(yearStart, startMonthFromBirth);
  const overlapEnd   = Math.min(yearEnd,   endMonthFromBirth);
  const overlapMonths = Math.max(0, overlapEnd - overlapStart);
  if (!overlapMonths) return 0;
  const monthlyIncome = person.income != null ? person.income : (state.finance.income || 0);
  const rate = (person.incomeRate ?? 67) / 100;
  return monthlyIncome * overlapMonths * (1 - rate);
};
```

- 数式: 「育休期間中の減少額（万円）= `monthlyIncome × overlapMonths × (1 − rate/100)`」
- `rate` の既定 67 → `1 − 0.67 = 0.33`（減少率 33%）。
- `offsetMonths` による育休開始タイミング:
  - `mom.leaveStart = 'after_maternity'` → 産後 2 ヶ月（産休相当）オフセットで育休開始。`'birth'`（出産月）は 0 オフセット。
  - `dad.leaveStart = 'after'` → 妻の育休が終わった後。`'birth'` は 0 オフセット。
- **年またぎ**: `yearStart = age × 12`、`yearEnd = yearStart + 12` の重なり月数で按分。出産月＝`age=0` の 1〜12 月目なので、カレンダー月と誕生月の乖離は考慮されない（=誕生月 `= 1月` と仮定）。
- **ボーナスは対象外**: 現実の雇用保険育休給付も標準報酬月額ベースで賞与を除外するため、この仕様は合理的。
- **旧データ互換（`parentalLeave` 未設定）** (`index.html:13053-13060`):
  ```js
  const maternityYears = (c.maternityMonths || 0) / 12;
  if (age < maternityYears) {
    const annualIncome = (state.finance.income || 0) * 12 + (state.finance.bonus || 0);
    costs.childcare += annualIncome * (1 - (c.maternityRate || 67) / 100);
  }
  ```
  - 旧形式は**年額に賞与を含めてしまっている**（`03-I02` 参照）。

### 2.4 賃貸（`index.html:13072-13086`）

```js
if (le.housingType === 'rent') {
  const r = le.rent;
  if (r.monthly) {
    const s = r.startYear ? parseInt(r.startYear) : new Date().getFullYear();
    const e = r.endYear ? parseInt(r.endYear) : 9999;
    if (year >= s && year <= e) {
      const yrs = year - s;
      const riseRate = (parseFloat(r.riseRate) || 0) / 100;
      const monthly = (parseFloat(r.monthly) || 0) * Math.pow(1 + riseRate, yrs);
      const admin = parseFloat(r.admin) || 0;
      const renewal = (year - s) % 2 === 1 ? (parseFloat(r.renewal) || 0) : 0;
      costs.mortgage += Math.round((monthly + admin) * 12 + renewal);
    }
  }
}
```

- 数式: `rent_annual(year) = (monthly × (1+riseRate)^(year−startYear) + admin) × 12 + renewalIfOddOffset`
- `admin` は家賃上昇率に追従しない（固定）。
- 更新料 `renewal` は `(year − startYear) % 2 === 1` の年のみ加算 → **startYear から見て奇数年目にのみヒット**。隔年の更新料は正しいが、一般的な 2 年契約の更新は「2 年目末・4 年目末」なので **`% 2 === 1` はオフセットが実質 1 年前倒し**になる（`03-I04` 参照）。
- 結果は `costs.mortgage` に加算される（賃貸も「住居費」として同じキーで計上）。

### 2.5 介護費（`index.html:13117-13120`）

```js
const ca = le.care;
if (ca.startYear && year >= parseInt(ca.startYear) && (!ca.endYear || year <= parseInt(ca.endYear))) {
  costs.care += (parseFloat(ca.monthlyFee) || 0) * 12;
}
```

- 数式: `care_annual = monthlyFee × 12`（フラット、固定）。
- 期間は `[startYear, endYear]`（両端 inclusive）。`endYear` 未指定時は開始以降ずっと。
- 一時費用（住宅改造・介護用ベッド等、生保文化センターで平均 47.2 万円）の計上口座はなし（`03-I01` 参照）。
- `careType`（`home` 等）はこの計算では未使用（UI ラベル用のみ）。

### 2.6 奨学金返済（`index.html:13124-13131`）

```js
(le.scholarships || []).forEach(sc => {
  if (!sc.monthlyPayment || !sc.startYear) return;
  const sy = parseInt(sc.startYear);
  const ey = sc.endYear
    ? parseInt(sc.endYear)
    : (sy + Math.ceil((parseFloat(sc.borrowedAmount) || 0) / (parseFloat(sc.monthlyPayment) || 1) / 12) - 1);
  if (year >= sy && year <= ey) {
    costs.scholarship += (parseFloat(sc.monthlyPayment) || 0) * 12;
  }
});
```

- 数式: `scholarship_annual = monthlyPayment × 12`
- `endYear` 未設定時の自動算出: `endYear = startYear + ⌈borrowedAmount / monthlyPayment / 12⌉ − 1`
  - **利子（第二種奨学金は有利子）を考慮していない**（`03-I03` 参照）。元本 ÷ 月次返済 ÷ 12 は単利 0% 換算で返済完了年を出す式。
- 単一の奨学金に対して `startYear` と `endYear` の**全期間一定の月返済額**を適用。定額返還方式のみサポートで、所得連動返還方式（JASSO「所得連動返還方式」）には非対応。

### 2.7 住宅ローン（`index.html:13087-13113`）

- 詳細はタスク 6（ⓓ）の対象。本監査ではハンドオフポイントのみ確認:
  - `calcMortgageSchedule()` が返す `schedule.get(year)` の `monthlyPayment × 12` を年額として使用。
  - 繰上返済 `prepay` は年内合計が残債の 50% 以上で一括扱い、未満で加算扱い（`index.html:13099-13110`）。
  - 家賃と同じ `costs.mortgage` キーに合算 → 後段のグラフ・解説で区別されない。

## 3. 標準との突合

### 3.1 幼稚園〜高校の学習費（`EDU_COST` vs 文科省）

- **標準**: 文部科学省「令和 5 年度 子供の学習費調査」（2024 年 12 月 25 日公表）。公立幼稚園 **184,646 円**、私立幼稚園 **347,338 円**、公立小学校 **336,265 円**、私立小学校 **1,828,112 円**、公立中学校 **542,475 円**、私立中学校 **1,560,359 円**、公立全日制高校 **597,752 円**、私立全日制高校 **1,030,283 円**（いずれも年額、学校教育費＋学校給食費＋学校外活動費の合計）。
  - 出典: 文部科学省「結果の概要-令和 5 年度子供の学習費調査」 <https://www.mext.go.jp/b_menu/toukei/chousa03/gakushuuhi/kekka/k_detail/mext_00002.html>
  - 出典: 文部科学省 PDF「令和 5 年度子供の学習費調査の結果を公表します」 <https://www.mext.go.jp/content/20241225-mxt_chousa01_000039333_1.pdf>
- **本コード**: 令和 3 年度ベースとコメントされている。

#### フェーズ別突合（本コード 万円/年 vs 令和 5 年度実績 万円/年換算）

| フェーズ | 区分 | 本コード | 令和 5 年度実績 | 差 |
|---|---|---|---|---|
| 幼稚園 | 公立 | **6** | 18.46 | **−12.46 万**（コード過少 67.5%） |
| 幼稚園 | 私立 | **17** | 34.73 | **−17.73 万**（コード過少 51.1%） |
| 小学校 | 公立 | **35** | 33.63 | +1.37 万（ほぼ一致） |
| 小学校 | 私立 | **167** | 182.81 | −15.81 万（コード過少 8.6%） |
| 中学校 | 公立 | **53** | 54.25 | −1.25 万（ほぼ一致） |
| 中学校 | 私立 | **143** | 156.04 | −13.04 万（コード過少 8.4%） |
| 高校（全日制） | 公立 | **51** | 59.78 | −8.78 万（コード過少 14.7%） |
| 高校（全日制） | 私立 | **105** | 103.03 | +1.97 万（ほぼ一致） |

**検算例（公立幼稚園）**: `18.46 − 6 = 12.46`、過少率 = `(18.46 − 6) / 18.46 = 12.46 / 18.46 = 0.6750 ≈ 67.50%`。

**検算例（全公立 幼〜高 合計）**:
- 本コード: `6×3 + 35×6 + 53×3 + 51×3 = 18 + 210 + 159 + 153 = 540 万円`
- 令和 5 年度: `18.46×3 + 33.63×6 + 54.25×3 + 59.78×3 = 55.38 + 201.78 + 162.75 + 179.34 = 599.25 万円`
- 差: `599.25 − 540 = 59.25 万円`（約 9.9% 過少）。
- 文科省の 15 年累計公表値「596 万円（すべて公立）」は端数丸めで整合。

**判定**: ⚠️ 公立幼稚園の過少幅が**著しく大きい**（67.5%）。コード注記「（無償化後）」は、2019 年 10 月からの幼児教育・保育無償化（3-5 歳、月上限 2.57 万円まで）を踏まえた調整と思われるが、**文科省調査は無償化後の令和 5 年度時点でも「学校外活動費を含めた保護者負担」は 18.46 万円/年**としており、本コードの 6 万円/年は学校外活動費（習い事等）を完全に除外している。UI の hint にその旨の明示がない（§6 `03-I05`）。

### 3.2 大学費用（`EDU_COST.university` vs 文科省・JASSO）

- **標準**:
  - 国立大学 2025 年度 標準額（国立大学等の授業料その他の費用に関する省令）: 入学金 **282,000 円**、授業料 **535,800 円/年**。4 年間総額 = `282,000 + 535,800×4 = 2,425,200 円 = 242.52 万円`。年額換算（入学金を 1 年目に加算せず平均化）= `2,425,200 / 4 / 10,000 = 60.63 万円/年`。
    - 出典: 文部科学省「国立大学等の授業料その他の費用に関する省令」 <https://laws.e-gov.go.jp/law/416M60000080016>
    - 出典: 文部科学省「国立大学と私立大学の授業料等の推移」 <https://www.mext.go.jp/b_menu/shingi/kokuritu/005/gijiroku/attach/1386502.htm>
  - 私立大学文系 4 年総額 ≈ **410 万円** → 102.5 万円/年（旺文社 2025 年度調査）。
  - 私立大学理系 4 年総額 ≈ **542 万円** → 135.5 万円/年（同上）。
    - 出典: 旺文社教育情報センター「2025 年度 大学の学費平均額」 <https://eic.obunsha.co.jp/educational_info/2025/2025%E5%B9%B4%E5%BA%A6-%E5%A4%A7%E5%AD%A6%E3%81%AE%E5%AD%A6%E8%B2%BB%E5%B9%B3%E5%9D%87%E9%A1%8D/>
- **本コード**: `national: 82` / `private_lib: 115` / `private_sci: 154`。
- **判定**: 国立は **過大（82 vs 60.63、+21.37 万/年 = 35.2%）**、私立文系・理系も **約 13% ずつ過大**。
  - 検算: `(82 − 60.63) / 60.63 = 21.37 / 60.63 ≈ 0.3524 ≈ 35.24%` 過大
  - 検算: `(115 − 102.5) / 102.5 = 12.5 / 102.5 ≈ 0.1220 ≈ 12.20%` 過大
  - 検算: `(154 − 135.5) / 135.5 = 18.5 / 135.5 ≈ 0.1365 ≈ 13.65%` 過大
- 差異の原因は以下の可能性:
  1. 「授業料＋入学金以外の諸費用（施設費・教材費・通学費）を含む学生生活費ベース」
  2. 教科書/下宿代等を含む日本政策金融公庫「教育費負担の実態調査」ベース
  3. 令和 3 年度以前の古い数字
- コメントに「文部科学省子供の学習費調査」としか書かれていないが、文科省子供の学習費調査は**大学を含まない**。出典が誤っているか追記漏れ（§6 `03-M01`）。

### 3.3 保育料（`EDU_COST.nursery` vs 自治体実績）

- **標準**: 認可保育園の保育料は **0-2 歳児のみ有料**（3-5 歳は 2019 年無償化）。住民税所得割額に応じた階層制で、自治体により上限も異なる。
  - 世田谷区・中央区などの東京 23 区: 0 歳児で月額 **2〜5 万円程度**（一般的な共働き世帯帯） → 年額 **24〜60 万円**。
  - 地方自治体: 月額 **2〜4 万円**（年額 **24〜48 万円**）程度。
  - 国全体の平均（2 歳児統計局調査）: **月額 37,755 円 ≈ 年額 45.3 万円**。
    - 出典: 住まいサーフィン「保育園の料金はいくら？」 <https://www.sumai-surfin.com/columns/mansion-knowledge/hoiku2>
    - 出典: 江東区「令和 6 年 4 月からの保育料の改定」 <https://www.city.koto.lg.jp/280308/hoikuryou/202404kaitei.html>
    - 出典: 川崎市「認可保育所等の利用者負担額（保育料）」 <https://www.city.kawasaki.jp/kodomo/category/267-2-7-0-0-0-0-0-0-0.html>
- **本コード**: `nursery: { public: 20, private: 45 }`（万円/年）。0-2 歳の 3 年間に毎年この額を計上。
- **判定**:
  - `public: 20` 万円/年 = 月 1.67 万円は**低所得〜中間層寄りの値**。都内共働きの平均像としてはやや低め。
  - `private: 45` 万円/年 = 月 3.75 万円は**認可外の平均価格帯**（月 5〜8 万円が相場）としては低い。
  - 所得階層・自治体・きょうだい割引などの**応能負担ロジックが一切未実装**。（§6 `03-I06`）。
  - 加えて 3-5 歳は無償化されているが、幼稚園（3-5 歳）には `kindergarten: { public: 6, private: 17 }` が当たる（≒保育所も同じ扱い）。UI 上「保育園派」の家庭は `nursery` を 0-2 歳に、3-5 歳は `kindergarten` として選ぶ想定だが、UI の誘導なし。

### 3.4 育休給付と収入減の算式（`calcLeaveReduction` vs 雇用保険）

- **標準**: 雇用保険の育児休業給付金は「休業開始時賃金日額 × 支給日数 × 67%（開始〜180 日）／50%（181 日目以降）」。
  - 出典: 厚生労働省 Q&A「育児休業等給付」 <https://www.mhlw.go.jp/stf/seisakunitsuite/bunya/0000158500.html>
  - 出典: 厚生労働省「育児休業給付金の拡充について」 <https://jsite.mhlw.go.jp/ehime-roudoukyoku/content/contents/002103176.pdf>
  - 2025 年 4 月からの **出生後休業支援給付金**（父母共に 28 日以上育休取得時）を合算すると、合計給付は **賃金日額 × 支給日数 × 80%**（180 日以内）まで拡充。
    - 出典: 契約Watch「育児休業給付金とは？支給額・上限・手取り 10 割」 <https://keiyaku-watch.jp/media/hourei/ikujikyugyo-kyufukin/>
- **本コード**: `1 − rate/100`（`rate` 既定 67）を**期間全体に一律適用**。
- **判定**: ⚠️ 期間分岐なし。既定 67% では「180 日目までは正確 (減少率 33%)、181 日目以降は 17% 分過少見積もり」（正しくは減少率 50% = 0.50 の減少）。
  - 検算（月収 30 万、育休 12 ヶ月、出産月から取得の場合）:
    - 正しい: 減少額 = `30 × 6 × (1 − 0.67) + 30 × 6 × (1 − 0.50) = 30 × 6 × 0.33 + 30 × 6 × 0.50 = 59.4 + 90.0 = 149.4 万円`
    - 本コード（67% 一律）: `30 × 12 × (1 − 0.67) = 30 × 12 × 0.33 = 118.8 万円`
    - 過少評価: `(149.4 − 118.8) / 149.4 = 30.6 / 149.4 ≈ 0.2048 ≈ 20.48%`
  - ユーザーは `incomeRate` を手動設定できるが、期間分岐は UI に存在しない。（§6 `03-I07`）

### 3.5 介護費（`care` vs 生保文化センター）

- **標準**: 生命保険文化センター「2024（令和 6）年度 生命保険に関する全国実態調査（2 人以上世帯）」
  - 月額 **平均 9.0 万円**（在宅 5.3 万円 / 施設 13.8 万円）
  - 介護期間 **平均 55.0 ヶ月**（4 年 7 ヶ月）
  - 一時費用 **平均 47.2 万円**（住宅改造・介護用ベッド等）
  - 介護総額の目安: `47.2 + 9.0 × 55.0 = 47.2 + 495.0 = 542.2 万円`
    - 出典: 生命保険文化センター「介護にはどれくらいの費用・期間がかかる？」 <https://www.jili.or.jp/lifeplan/lifesecurity/1116.html>
- **本コード**: `monthlyFee × 12` フラット。`monthlyFee` はユーザー入力（既定なし）。期間もユーザー入力（`startYear` / `endYear`）。
- **判定**:
  - 数式自体は単純で妥当。**ただし一時費用（平均 47.2 万円）を入れる場所がない**。ユーザーは `state.expenses[]`（計画一時支出）で別途登録するか、初年度の `monthlyFee` を上乗せして擬似的に表現する必要があるが、UI 側の誘導は確認できず。
  - 設定の既定値ヘルパーがないため、ユーザーが UI で「月 9 万／4.6 年」のような実態値を思いつかない限り**過少推計になりやすい**（§6 `03-I01`）。
  - 在宅 / 施設の区分（`careType`）は UI 上選択できるが、計算では使われない（§6 `03-M02`）。

### 3.6 奨学金返済（`scholarship` vs JASSO）

- **標準**: 日本学生支援機構（JASSO）第二種奨学金（有利子）は **定額返還方式のみ**。貸与月額 2 万〜12 万円（1 万円刻み）、貸与期間は 4 年（大学）。返還月額は貸与総額で決定。
  - 例（JASSO 公式シミュレーション例、2025 年 3 月貸与終了・月 5 万円 4 年・貸与総額 240 万円）:
    - 返還期間: 15 年（180 ヶ月）
    - 利率: 1.641%
    - 毎月返還額: **15,157 円**
    - 総返還額: `15,157 × 180 = 272.8 万円`（元本 240 + 利息 32.8）
  - 出典: JASSO「返還のてびき 2025 年度在学者用」 <https://www.jasso.go.jp/shogakukin/moshikomi/zaigaku/tebiki/__icsFiles/afieldfile/2025/03/12/annnaitou.pdf>
  - 出典: JASSO「第二種奨学金の貸与月額」 <https://www.jasso.go.jp/shogakukin/about/taiyo/taiyo_2shu/kingaku.html>
- **本コード**:
  - 年額 = `monthlyPayment × 12`（定額返還はこの式で正確）。
  - `endYear` 未設定時の自動計算: `sy + ⌈borrowedAmount / monthlyPayment / 12⌉ − 1`。
- **判定**:
  - 返済期間の自動計算は**利息を無視**するため、第二種奨学金（有利子）では**返済完了が早くなりすぎる**。
  - 検算（貸与総額 240 万、月返済 1.5 万）:
    - 本コード: `sy + ⌈240 / 1.5 / 12⌉ − 1 = sy + ⌈13.33⌉ − 1 = sy + 14 − 1 = sy + 13`（14 年間返済）
    - JASSO 現実（月 5 万 4 年貸与総額 240 万の例）: 毎月 15,157 円 = 1.5157 万円、15 年間。本コードの貸与 240/月 1.5 万前提と近いが、**コードは「月 1.5 万円返済で 14 年で完済」と計算**。実際は利息のせいでそうはいかない。
  - ユーザーが `endYear` を手動で入れれば問題にならないが、UI の `calcScholarshipEndYearPreview` が**利息無視の年数を提示**する点が誤誘導の温床（§6 `03-I03`）。
  - 所得連動返還方式（卒業後の給与に応じて返還月額が変動、返済期間は最長 20 年）は**サポート外**。JASSO の仕様では第一種奨学金でのみ選択可能。ミニマル実装としては許容範囲（§6 `03-M03`）。

### 3.7 賃貸費用（`calcRentResult` の 10/30 年と `calcLECostByYear` の年次）

- **標準**: 家賃は月額 × 12 + 共益費 × 12 + 更新料（2 年契約の場合、24 ヶ月ごと）。敷金礼金は初期一時費用で分離。
- **本コード**:
  - UI プレビュー (`calcRentResult`) と年次計算 (`calcLECostByYear`) で **更新料の発生年が異なる**。
    - UI プレビュー: `for i in 0..y: if i % 2 === 1 ...` → i=1,3,5,...（2 年目末、4 年目末、6 年目末）に更新料加算 → 正しい。
    - 年次計算: `(year − s) % 2 === 1` → s+1, s+3, s+5,... で加算 → **入居 1 年目末ではなく、2 年目中の暦年にヒット**。契約更新は「2 年居住後」なので 2 年目末 = 2 年目の暦年の末尾だが、`year − s = 1` は「入居 1 年後の暦年」に相当し **1 年ずれの感覚**で加算されている。
  - 実質は「入居年から奇数年後」に計上。契約更新月によっては正確ではない（初年度 4 月入居なら 2 年目の 4 月に更新料が発生 → `year − s = 1` の暦年内に含まれるのでたまたま一致）。
  - 敷金・礼金は全く計算されない（§6 `03-M04`）。
  - `admin`（管理費）は家賃上昇率の対象外（固定）。実態としては管理費も値上がりしうるが、通常は長期据え置きなので許容。
- **判定**: ✅ 近似としては十分だが、更新料タイミングは要確認。

### 3.8 教育費に対する物価スライドなし

- **標準**: 国立大学授業料は 2004 年 535,800 円から据え置き（2025 年度も同額）だが、私立大学授業料は**年率 1-2% で緩やかに上昇**している。文科省推移資料で確認可能。
- **本コード**: テーブル値は絶対額。`year` パラメータを受けているにもかかわらず、将来年も現在の万円額がそのまま使われる。インフレ率（`finance.inflationRate`、既定 2%）は適用されない。
- **判定**: ⚠️ 20 年後の子どもの教育費は少なくとも 1.02^20 ≈ 1.486 倍になる想定が自然だが、本コードでは 0 年後と同額。
  - 検算（私立大学理系 154 万、子ども現在 0 歳、大学入学 18 年後）:
    - 本コード: 154 × 4 = 616 万円（名目）
    - 2%/年 インフレ反映: `154 × ((1.02^18 + 1.02^19 + 1.02^20 + 1.02^21)) ≈ 154 × (1.4282 + 1.4568 + 1.4859 + 1.5157) = 154 × 5.8865 ≈ 906.5 万円`（名目）
    - 過少評価（18 年後 4 年累計）: `(906.5 − 616) / 906.5 ≈ 0.3204 ≈ 32.04%`
  - これは `02-I01`（現役期支出にインフレ未適用）と同じ根本問題だが、ライフイベント費用は現役期の最大支出要因なので影響が大きい（§6 `03-I08`）。

### 3.9 他項目との二重カウント懸念

- **住宅頭金**: `state.expenses[]`（一時支出）に「住宅購入頭金」をユーザー登録した場合と、`le.mortgage.downPayment` を設定した場合の両方ありえる。`calcLECostByYear` の `mortgage` は `calcMortgageSchedule` の月次返済ベースなので `downPayment` は含まれないが、`downPayment` 自体は別経路でキャッシュフローに反映される必要がある。タスク 6（ⓓ）で詳細確認予定（§6 `03-M05`）。
- **介護一時費用**: ユーザーが `state.expenses[]` で手動登録した場合、`care.monthlyFee` と一緒に計上されるのは正しい挙動だが、UI の注意喚起なし。

## 4. 仮定・制約

1. **教育費はフェーズ内で一定**: 同じフェーズの全年齢で同じ額。高 3 受験年の塾代・入学金スパイクは平準化されている。
2. **`PHASE_AGES` のレンジは固定**: 留年・飛び級・中退は表現不可。高校中退（15-16 歳で終わり）や大学院進学（22 歳以降も継続）は非対応。
3. **子どもフェーズの選択肢**: 各フェーズ独立（`c.phases.nursery`, `c.phases.university` 等）。フェーズ間の進路依存（例: 中高一貫校ルート）の専用カテゴリは存在しない。選択肢は「公立 or 私立」のみで、中高一貫の前半（中学校）が私立 = 私立中学校で表現する必要がある。
4. **誕生月は 1 月と仮定**: 育休重なり計算で `age × 12` から月を数えるため、**誕生月に関係なく年初を基準**にしている。
5. **`incomeRate` 既定 67 は期間全体一律**: 181 日目以降の 50% ルール非対応（§3.4）。
6. **育休中のボーナス**: `calcLeaveReduction` は月収 × 月数のみを使うため、**ボーナスの無給分は計上されない**。実際の育休では賞与支給がない/減額されることが多い。
7. **奨学金は定額返還のみ**: 所得連動返還方式・繰上返還・猶予期間は非対応。
8. **介護費はフラット**: 要介護度の進行（要介護 1 → 要介護 5）による段階的上昇は表現不可。
9. **賃貸更新料は 2 年周期固定**: 1 年契約・3 年契約などの非標準周期には対応しない。
10. **インフレ率非適用**: 教育費テーブル・保育料・介護費・奨学金返済・家賃基礎部分は現在価格（`riseRate` だけは家賃のみ対象）。

## 5. エッジケース

1. **`year < c.birthYear`（未出生）**: `age < 0` で `return` → 0（`index.html:13019`）。正しい。
2. **`year = c.birthYear`（出産年）**: `age = 0` → nursery フェーズヒット → 保育費 20 万円計上。**実際には出産年の途中から保育園に入園するのが普通**（育休中は不通園）。`state.lifeEvents.children[].phases.nursery = 'none'` 的なオプションはなく、保育費と育休収入減が**同年に二重計上**される可能性あり（§6 `03-I09`）。
3. **`parentalLeave` 両親とも `leaveMonths = 0`**: `calcLeaveReduction` が 0 を返すので問題なし。
4. **`mom.leaveStart = 'after_maternity'` & `mom.leaveMonths = 0`**: `calcLeaveReduction` 内で `if (!leaveMonths) return 0` → オフセットの 2 ヶ月分も 0 にできる。逆に `leaveMonths = 1` かつ `leaveStart = 'after_maternity'` なら 産後 2 ヶ月 + 育休 1 ヶ月 = 計 3 ヶ月分のうち育休 1 ヶ月分だけ計上（産休 2 ヶ月分の減収は計上されない）。
5. **父親 `leaveStart = 'after'` & 母親未設定**: `(pl.mom?.leaveMonths || 0)` → 0 → 父親 offset = 0。父が出産月から育休開始扱いになる。UI の想定と異なる可能性あり。
6. **`c.maternityMonths` 旧データ**: `c.parentalLeave` があるとスキップ、ないと**賞与を含む年収ベース**で計算（§2.3）。旧データ移行時に過大推計（§6 `03-I02`）。
7. **賃貸 `startYear` 未設定**: `new Date().getFullYear()` を既定使用 (`index.html:13076`)。シミュレーション実行時の年に依存するため、**翌年実行すると 1 年ズレる**。保存時に明示的に `startYear` を設定するのが望ましい（§6 `03-M06`）。
8. **奨学金 `startYear` のみ設定・`endYear` なし・`borrowedAmount` 0**: `ey = sy + ⌈0/monthly/12⌉ − 1 = sy − 1` → `year >= sy && year <= sy − 1` は常に false → **一切返済計上されない**。`borrowedAmount = 0` で返済だけ続くケース（残債未記録）が正しく扱えない（§6 `03-M07`）。
9. **奨学金 `monthlyPayment = 0`**: `if (!sc.monthlyPayment || !sc.startYear) return` でスキップ。正しい。
10. **`ca.startYear` 空文字**: `ca.startYear && year >= parseInt('')` → `parseInt('') = NaN` → `year >= NaN` は false → スキップ。正しい。
11. **`ca.endYear` 空 & `ca.startYear` 未来**: 開始年未到達で 0、開始年以降は永久に月額加算。`endYear` 未設定時の無限介護は表現できるが、UI で明示するのが望ましい。
12. **`housingType = 'none'`**: `if (le.housingType === 'mortgage' || !le.housingType)` 経路もスキップされ、賃貸経路も `'rent'` でなければ通らない → 0。正しい。
13. **同年に複数子の育休重なり**: 子 A 育休 12 ヶ月 + 子 B 育休 12 ヶ月 が同年に発生 → それぞれ `calcLeaveReduction` で独立計算され**両方加算される**。1 年に 24 ヶ月分の減収 → 不自然（雇用保険は同時には 1 つの育休しか取得不可）。`03-I10` 参照。
14. **`yearsElapsed = 0`（シミュレーション初年度）**: 育休が出産年から開始で現在年 = 出産年なら 0〜12 ヶ月すべて対象 → 12 ヶ月分減収計上。正しい。

## 6. 検出された問題（深刻度付き）

### 🔴 Critical

*（なし。本関数はシンプルな集計器で、**計算そのものには設計上の単純バグ**は見つからなかった。下記 `03-I01`〜`03-I10` は「値の妥当性」「欠けた分岐」「二重計上」でありクリティカルに達しない。`02-I01`（インフレ未適用）と同じ構造的欠陥は `03-I08` として Important に格下げ）*

### 🟡 Important

- **`03-I01` 介護一時費用（平均 47.2 万円）を計上する場所がない**
  - `care` には `monthlyFee` しかなく、生保文化センター 2024 年度調査の「一時費用 平均 47.2 万円」を入れる field が存在しない。ユーザーは `state.expenses[]` で手動登録する必要があるが、UI 誘導もヘルパーもない。
  - 数値インパクト（介護総額 542.2 万円想定のうち、47.2 万円が欠落 → 8.71% 過少）:
    - 正しい: `47.2 + 9.0 × 55.0 = 47.2 + 495.0 = 542.2 万円`
    - 本コード: `9.0 × 55.0 = 495.0 万円`
    - 過少率: `47.2 / 542.2 ≈ 0.0871 ≈ 8.71%`
  - **修正方針**: `care.oneTimeFee` を追加し、`startYear` 年度のみ `oneTimeFee` を `costs.care` に加算。または UI で「一時費用 47.2 万円を `state.expenses[]` に自動登録するボタン」を用意。

- **`03-I02` 旧データ互換経路（`maternityMonths`）は賞与込みで過大推計**
  - `index.html:13053-13060` の fallback パスは `annualIncome = income×12 + bonus` に `(1 − rate)` を掛ける。一方 `calcLeaveReduction` 経路は月収のみベース（賞与除外）。
  - 数値インパクト（月収 30 万、ボーナス年 80 万、育休 12 ヶ月、rate 67%）:
    - 新パス: `30 × 12 × 0.33 = 118.8 万円`
    - 旧パス: `(30 × 12 + 80) × 0.33 = 440 × 0.33 = 145.2 万円`
    - 差: `145.2 − 118.8 = 26.4 万円`（旧パスが 22.2% 過大）
  - **修正方針**: 旧パスも `state.finance.income × 12 × (1 − rate)` に揃える（賞与を除外）。

- **`03-I03` 奨学金の返済終了年自動計算が利息を無視**
  - `ey = sy + ⌈borrowedAmount / monthlyPayment / 12⌉ − 1` は利率 0% 前提。第二種奨学金（JASSO 2025 年 3 月貸与終了利率 1.641%）では返済期間が伸びる。
  - 数値インパクト（貸与総額 240 万、月返済 1.5157 万 = JASSO 実例、利率 1.641%）:
    - JASSO 実際: 15 年（180 ヶ月）
    - 本コード: `⌈240 / 1.5157 / 12⌉ = ⌈13.20⌉ = 14 年`
    - 逆方向の例（月 1.5 万、貸与 240 万、同利率）: 本コードは 14 年、JASSO 実態は **約 15.1 年**（総返済額約 272 万）で、年次返済額を過大に見せる（毎月を高く逆算）か終了年を早く見せるかのどちらか。
  - 総返済額を捨てて「毎月返済額 × 12」だけで年次計算している設計上、**総返済額（元本＋利息）と月返済額の整合は取れていない**。UI で月返済額を正確に入力してもらえば問題ないが、`calcScholarshipEndYearPreview` が利息無視の終了年を自動セットする挙動（`index.html:13359-13361`）がユーザーを誤誘導する。
  - **修正方針**: `scType` に `'jasso_1'` / `'jasso_2'` を持つので、`jasso_2` は利率補正係数を掛けて終了年を算出（例: 1.641% × 15 年で月返済 15,157 円相当に合わせる）。または「終了年は手動入力推奨」と UI に表示。

- **`03-I04` 賃貸更新料の発生年タイミングが入居 1 年目末から**
  - `(year − s) % 2 === 1` は入居年 s を 0 とすると s+1, s+3, ... が対象。通常 2 年契約の賃貸では「入居日から 2 年後」に更新料が発生するので、`s+1` 暦年にヒットすると**1 年前倒し**になる可能性が高い。
  - 数値インパクト（月 10 万・更新料 10 万、5 年居住）:
    - 本コード: s+1 (10 万)、s+3 (10 万) → 合計 20 万
    - 実態（s 年 4 月入居・2 年契約）: s+2 年 4 月と s+4 年 4 月 → 該当暦年は s+2, s+4 → 合計 20 万（金額は一致、年がずれる）
  - 影響は金額合計ではなく**発生年の 1 年シフト**。ダッシュボードの年次赤字判定が 1 年ずれる。
  - **修正方針**: `(year − s) % 2 === 1` を `(year − s) >= 2 && (year − s) % 2 === 0` に変更。または `(year − s − 1) % 2 === 1 && year > s` で s+2, s+4 にマッチさせる。

- **`03-I05` 公立幼稚園の 6 万円/年は学校外活動費を除外し過少**
  - 文科省令和 5 年度調査「公立幼稚園 18.46 万円/年（学校教育費 6.19 + 給食費 1.98 + 学校外活動費 10.29）」に対し、本コードは `public: 6` 万円/年。
  - 数値インパクト（幼稚園 3 年間、公立）:
    - 本コード: `6 × 3 = 18 万円`
    - 文科省: `18.46 × 3 ≈ 55.38 万円`
    - 過少: `(55.38 − 18) / 55.38 ≈ 0.6749 ≈ 67.49%`
  - 学校教育費相当だけを見ているコードは、**無償化後の「実質保護者負担」を 3 分の 1 未満に見積もる**。
  - **修正方針**: `kindergarten: { public: 18, private: 35 }` に更新。コード注記の「（無償化後）」が誤誘導なので削除し「学校外活動費込み」と明記。

- **`03-I06` 保育料の所得連動・自治体差・きょうだい割引が未実装**
  - `EDU_COST.nursery: { public: 20, private: 45 }` 万円/年のフラット額のみ。実態は住民税所得割額に応じた階層制で、共働き世帯年収 800〜1000 万では公立でも月 4〜5 万円（年 48〜60 万円）になるケースが多い。
  - 数値インパクト（東京 23 区・世帯年収 800 万・0 歳児公立保育園）:
    - 本コード: 20 万円/年
    - 実態（江東区 D11 階層相当）: 月 4〜5 万円 → 年 48〜60 万円
    - 過少: `(54 − 20) / 54 ≈ 0.6296 ≈ 62.96%`（中央値 54 万で試算）
  - **修正方針**: `state.finance.income` を引数にして所得階層を判定する関数 `calcNurseryFee(yr, age)` を追加。最小実装でも「高所得想定 / 中所得想定 / 低所得想定」3 段階選択を UI に追加。

- **`03-I07` 育休給付の期間分岐（67% / 50%）が未実装**
  - `calcLeaveReduction` は `rate` 一律。開始 181 日目以降は **減少率が 50%（1 − 0.5）になる**べき（雇用保険法の定め）。
  - 数値インパクト（月収 30 万、育休 12 ヶ月、出産月から取得）:
    - 正しい: `30 × 6 × (1 − 0.67) + 30 × 6 × (1 − 0.50) = 30 × 6 × 0.33 + 30 × 6 × 0.50 = 59.4 + 90.0 = 149.4 万円`
    - 本コード（67% 一律）: `30 × 12 × 0.33 = 118.8 万円`
    - 過少: `(149.4 − 118.8) / 149.4 ≈ 0.2048 ≈ 20.48%`
  - **修正方針**: `calcLeaveReduction` 内で overlap を「最初の 6 ヶ月」と「それ以降」で分けて `1 − 0.67` / `1 − 0.50` を使い分ける。2025 年 4 月制度変更（出生後休業支援給付金 +13% 合計 80%）への対応は任意。

- **`03-I08` 教育費・保育料・介護費・家賃にインフレが適用されない（`finance.inflationRate` 未使用）**
  - `02-I01` と同型問題。ライフイベント費用の大部分を占める項目が**名目固定**で、20 年後の大学入学時も現在の 154 万円/年（私立理系）のまま計算される。
  - 数値インパクト（私立理系 154 万、子ども 0 歳、大学入学 18 年後、インフレ 2%）:
    - 本コード: `154 × 4 = 616 万円`（名目）
    - 2% インフレ適用: `154 × (1.02^18 + 1.02^19 + 1.02^20 + 1.02^21) = 154 × (1.4282 + 1.4568 + 1.4859 + 1.5157) = 154 × 5.8865 ≈ 906.5 万円`（名目）
    - 過少: `(906.5 − 616) / 906.5 ≈ 0.3204 ≈ 32.04%`
  - 国立大学授業料は 2004 年から据え置きのため、「国立は名目固定」の方が現実に近いという反論はあるが、私立・保育料・家賃（`riseRate` は別経路）・介護費は**緩やかに上昇**している。
  - **修正方針**: `02-I01` と一体で、`getExpenseForYear` 系の修正と同じ方針でスライドを適用（またはユーザーに教育費インフレ率を別途入力させる）。タスク 10（ⓐ 統合シミュレーション）で一括対応を検討。

- **`03-I09` 出産年（`age = 0`）に育休収入減と保育料の両方が計上される可能性**
  - 出産年は `age = 0` で `nursery` フェーズに入り、`public: 20` 万円が毎年計上される（出産年含む 3 年間）。同時に同年に育休収入減も計上される。
  - 実態では出産年は育休中で保育園には通わないので、`nursery` 費用は**発生しない**はず。
  - 数値インパクト（公立保育園、出産年の重複）: 20 万円の過大計上。
  - **修正方針**: 育休期間と保育園通園期間の**排他ロジック**を入れる。例: `pl.mom.leaveMonths` と `pl.dad.leaveMonths` の最大が 12 ヶ月なら出産年の保育費を 0 にするなど。

- **`03-I10` 年子・双子の育休重複時に同年 24 ヶ月分の減収が計上される**
  - 子 A（birthYear=2026, leaveMonths=12）と子 B（birthYear=2027, leaveMonths=12）が登録されていると、2027 年は子 A の育休 12 ヶ月（age=1 → yearStart=12, yearEnd=24 に対し子 A の leave は 0-12 月 → 重なり 0、大丈夫）実際は `forEach` で独立計算され、子 B の age=0 の 12 ヶ月がフル重なり。問題は**双子**の場合で、同じ `birthYear` の子が 2 人いれば**両方で 12 ヶ月分ずつ = 24 ヶ月分減収が発生**する。雇用保険は同時に 1 つの育休しか取得できないため、これは過大。
  - 数値インパクト（双子、月収 30 万、rate 67%、双方で 12 ヶ月育休）:
    - 本コード: `30 × 12 × 0.33 × 2 = 237.6 万円`
    - 実態: `30 × 12 × 0.33 × 1 = 118.8 万円`（1 人分のみ）
    - 過大: `(237.6 − 118.8) / 118.8 = 1.0` = **100% 過大（2 倍）**
  - **修正方針**: 同じ `birthYear` の子は育休減収を 1 回分だけ計上するよう `children` を `reduce` で集約する。

### 🟢 Minor

- **`03-M01` 教育費テーブルの出典コメントが不正確**
  - `EDU_COST` のコメント「文部科学省子供の学習費調査 令和3年度ベース」は、同調査が**幼稚園〜高校のみ**を対象で大学・保育園を含まないため、`nursery` と `university` の出典としては誤り。令和 3 年度 → 令和 5 年度の更新も未反映。
  - **修正方針**: コメントを「幼稚園〜高校: 文科省子供の学習費調査（令和 5 年度）、保育料: 自治体参考値、大学: 文科省授業料標準額・旺文社調査ベース」に書き換え。

- **`03-M02` `care.careType`（在宅 / 施設）が計算に反映されない**
  - UI では `home` / `facility` 等を選べるが、`calcLECostByYear` は `careType` を参照せず `monthlyFee` のみ使う。生保文化センター平均は在宅 5.3 万円・施設 13.8 万円と **2.6 倍の差**がある。
  - **修正方針**: `careType` に応じた既定値を UI helper として提示。または `monthlyFee` 未設定時の既定値を `careType` で分岐。

- **`03-M03` 所得連動返還方式（JASSO 第一種）非対応**
  - JASSO 第一種（無利子）は所得連動返還方式を選択可能。返還月額は課税対象所得に応じて変動し、期間も最長 20 年まで伸びる。本コードは定額のみ。
  - 影響は小さく、ユーザーが手動で「平均的な月返済額」を入れれば近似可能。

- **`03-M04` 賃貸の敷金・礼金・仲介手数料が計算に入らない**
  - `calcLECostByYear` の rent 経路は月額・共益費・更新料・賃料上昇率のみ。初期一時費用（敷金 1〜2 ヶ月、礼金 1 ヶ月、仲介手数料 1 ヶ月、保証料）は計上されず、ユーザーが `state.expenses[]` に手動登録する必要がある。
  - 影響: 月 10 万円家賃で初期費用 40〜50 万円（敷金 20 + 礼金 10 + 仲介 10 + 保証料 5 等）が抜ける。10 年居住すると家賃総額 1,200 万の中で 3〜4% の過少。
  - 優先度は低いが、UI に「敷金・礼金」欄の追加が望ましい。

- **`03-M05` 住宅頭金 `downPayment` と `state.expenses[]` の二重カウント懸念**
  - `le.mortgage.downPayment` を設定した場合、年次キャッシュフローに反映させるには別経路が必要。ユーザーが `state.expenses[]` に同じ金額を手動登録していると二重カウントになる。タスク 6（ⓓ）で住宅ローン監査時に要検証。

- **`03-M06` 賃貸 `startYear` 未設定時の `new Date().getFullYear()` フォールバック**
  - 実行時の年に依存するため、保存データを翌年開く度に家賃開始年が 1 年スライドする。データの再現性が損なわれる。
  - **修正方針**: 保存時に明示的な `startYear` を設定する。未設定なら UI で必須入力にする。

- **`03-M07` 奨学金 `borrowedAmount = 0` & `endYear` 未設定で一切計上されないバグ**
  - §5 エッジケース 8 参照。「残債未記録で月返済だけ記録したい」ケースで詰む。
  - **修正方針**: `borrowedAmount <= 0` かつ `endYear` 未設定なら警告し、`endYear` を必須にする。

- **`03-M08` 賃貸と持ち家ローンが同じ `costs.mortgage` キーに集約され内訳が分からない**
  - 後段のグラフ・解説（`index.html:12181-12189`）で `mortgage` ラベルは「住宅ローン・家賃」と併記されているので UI 側では問題ないが、API 的には分離したほうが再利用性高い。
  - **修正方針**: `costs.housing` を新設し、内訳を `housing: { rent, mortgage }` に分ける。

## 7. 結論

- **信頼度**: ❌ 要対応
- **一言サマリー**: `calcLECostByYear` の**骨格の集計ロジックは単純で健全**（カテゴリ別加算・ゲート opts・期間判定）だが、(1) 教育費テーブル値が文科省令和 5 年度調査と比べ**公立幼稚園で 67% 過少、国立大学で 35% 過大**というキャリブレーションのずれ、(2) 育休給付の**67%/50% 期間分岐未実装**で長期育休時に 20% 過少、(3) 介護費の**一時費用 47.2 万円を入れる場所がない**、(4) 保育料の**所得連動が未実装で都内共働き世帯で 60% 以上過少**、(5) 奨学金返済期間が**利息無視で 1 年程度短く見積もる**、(6) 出産年の育休 + 保育費重複・双子の育休 2 重計上、(7) 教育費・介護費に**インフレが適用されず** 20 年後の私立理系大学費用を 32% 過少推計、という複数の実態乖離が直接的に年次コストに 10〜60% 級のバイアスを与える。
- **信頼度 ❌ 要対応 の根拠**: Important 10 件のうち `03-I01`（介護一時費用欠落）・`03-I05`（幼稚園公立過少 67%）・`03-I06`（保育料所得連動欠落）・`03-I07`（育休期間分岐欠落）・`03-I08`（教育費インフレ未適用）・`03-I09`（出産年保育費重複）・`03-I10`（双子育休 2 倍）は**標準の FP 相場から単独で 20% 以上の乖離**を生み、ユーザーが絶対額を信頼してプランを立てると**ライフイベント赤字年の発生時期予測を 1〜3 年前倒しまたは後ろ倒し**してしまうレベル。Critical 0 件なのは「設計上の単純バグ（Task 3 の `02-C01` 型）」が見つからなかったためだが、**複数の Important が合算されて年次コストの信頼性を損なう**点で ❌ とした。
- **申し送り**:
  - `03-I08`（教育費・介護費インフレ未適用）はタスク 10（ⓐ 統合シミュレーション監査）に `02-I01` と束ねて持ち込む。`calcIntegratedSim` で annualCosts にインフレ係数を掛ける最小侵襲修正を検討。
  - `03-I01`・`03-I05`・`03-I06`・`03-I07` は UI + 計算の両方に手が入るため、**フェーズ 3（設計修正）で 1 スプリント相当**の作業を要する。
  - `03-M05` は**タスク 6（ⓓ 住宅ローン監査）で要検証**。`le.mortgage.downPayment` がキャッシュフローでどこにヒットしているかを確認すること。
  - `03-I09` / `03-I10`（重複計上系）は実データで再現するには子ども複数登録が必要。**タスク 11（シナリオ B サニティウォークスルー）で 0 歳児 + 双子シナリオを試して挙動を確認**する。
  - 数値キャリブレーションは令和 5 年度最新データを反映すべき。`EDU_COST` のアップデートは 1 コミットで完結するので、フェーズ 3 の先頭で処理すれば以降の監査ノイズが減る。
